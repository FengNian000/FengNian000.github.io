单调队列

  • 单调队列使用双端队列实现。
  • 该队列具有单调性，可单调递增，递减，不递增，不递减。
  • 维护：每个新元素从队尾进入队列时都会与队尾相比较，然后把破坏单调性的队尾弹出，让新元素进入从而维护单调性

P1886 滑动窗口 /【模板】单调队列

区间RMQ问题，当然是用ST表来写啦，咳咳，我们可以使用两个单调队列分别维护区间 \([i, i + k - 1]\) 最大值和最小值，每压入新元素之前删去过期元素和破坏单调性的元素即可

#include <bits/stdc++.h>
#define N 1000010
using namespace std;

int n, k;
int ans1[N], ans2[N];
struct node {
    int tim, dat;
}; // 用入队时间和数据大小来维护

deque<node> q1; // 维护最小
deque<node> q2; // 维护最大

int main() {
    scanf("%d %d", &n, &k);
    for(int i = 1, tmp; i <= n; ++i) {
        scanf("%d", &tmp);
        while(!q1.empty() && q1.back().dat > tmp) q1.pop_back(); // 维护区间递增
        q1.push_back({i, tmp});
        while(!q1.empty() && q1.front().tim < i - k + 1) q1.pop_front(); // 删除过期窗口

        while(!q2.empty() && q2.back().dat < tmp) q2.pop_back(); // 维护区间递减
        q2.push_back({i, tmp});
        while(!q2.empty() && q2.front().tim < i - k + 1) q2.pop_front(); // 删除过期窗口

        ans1[i] = q1.front().dat, ans2[i] = q2.front().dat;
    }
    for(int i = k; i <= n; ++i) printf("%d ", ans1[i]);
    printf("\n");
    for(int i = k; i <= n; ++i) printf("%d ", ans2[i]);
    return 0;
}

单调队列优化DP

无论是单调队列优化还是单调栈优化其实本质就是借助其单调性来及时排除不可能的决策，保持待选集合的高度有效性和秩序性，举个栗子：

现有长度为 \(n\) 的序列， 从中选出长度为 \(m\) 子序列使得序列和最大（n, m \(\leq 3*10^5\) ）

显然我们可以轻松写出转移 \(ans = \max\limits_{1 \leq i \leq n}\{s_i - \min\limits_{i - m \leq j \leq i - 1}\{s_j\}\}\)

同样很显然的是该做法是 \(O(n^2)\) 的，现在考虑如何优化，对于每一个 \(i\) 都会遍历 \(j \in [i - m, i - 1]\)，对于 \(i + 1\) ， \(j\) 的取值会向后移动一位，此时意味着第 \(i - m\) 个决策会过期，同时第 \(i\) 个状态会被加入，看到这其实和单调队列板子就有异曲同工的地方了，删去过期决策和不满足的决策，加入新的且满足条件的决策，这不就和板子一样了吗，板子里我有说

每压入新元素之前删去过期元素和破坏单调性的元素即可

对于形如 \(f_i = \min/\max\{f_j + b_j + a_i\} \bigcup L(i) \leq j \leq R(n)(LR为窗口)​\) 其中与 \(i​\) 和 \(j​\) 有关的项 \(a_i​\) 和 \(b_j​\) 相互独立（对于形如 \(a_i \times b_j​\) 这种不相互独立的情况会在斜率优化中介绍），如果单纯的循环 \(i​\) 和 \(j​\) 复杂度是 \(O(n^2)​\)，但如果使用单调队列优化复杂度可降低到 \(O(n)​\)

为什么单调队列能优化

上文提到，每次 \(i​\) 移动一位之后一部分 \(j​\) 的决策与 \(i - 1​\) 时重合，

如图，\(i1\) 的决策 \(j1\) 和 \(i2\) 的 \(j2\) 中有一部分重叠，对于这些窗口，处理决策时有两种情况： - 随着窗口的移动被排除掉（\(j1\) 左侧部分） - 随着窗口移动未被排除，可能还会被使用（\(j1\) 与 \(j2\) 重叠部分）

这时使用单调队列就可以统一处理这些决策，优化掉内循环 \(j\) 从而将复杂度降到 \(O(n)\)

如何优化

还是这个方程 \(f_i = \max\{f_j + b_j + a_i\}\) ，\(i\) 是同时枚举的，所以可以拎出 \(a_i\) 变成 \(f_i = \max\{f_j + b_j\} + a_i\)，现在只要在窗口区间内找出 \(max\{f_j + b_j\}\) 即可找出此时的最优解，不妨记 \(x_j = f_j + b_j\) 然后用单调队列维护 \(x_j\) 递减，每次决策时只需要取出队首就可以求得 \(f_i\)。在 \(i\) 移动时，它前面的每个状态都最多入队出队一次，最终复杂度为 \(O(0)\)

练习

P2627 [USACO11OPEN]Mowing the Lawn G

先设计状态 \(f_i\) 为前 \(i\) 头牛的最大效率之和，方程为 \[f_i = \max\{f_{j - 1} + \sum\limits_{k = j + 1}^{i}{E_k}\} \bigcup (i - k \leq j \leq i)\] 求和部分可以用前缀和优化 \[f_i = \max\{f_{j - 1} + sum_i - sum_j\} \bigcup (i - k \leq j \leq i)\] 拎出 \(sum_i\)

\[f_i = \max\{f_{j - 1} - sum_j\} + sum_i \bigcup (i - k \leq j \leq i)\]

最后用单调队列维护 \(f_{j - 1} - sum_j\) 即可

#include <bits/stdc++.h>
#define N 100010
#define LL long long
using namespace std;

int n, k;
LL f[N], sum[N];

struct node{int tim; LL dat;};

deque<node>q;

int main() {
    scanf("%d %d", &n, &k);
    for(int i = 1;i <= n; ++i) {
        scanf("%d", &sum[i]);
        sum[i] = sum[i - 1] + sum[i];
    }

    q.push_back({0, 0});//压入一个0防止报错

    for(int i = 1;i <= n; ++i) {
        while(!q.empty() && q.back().dat <= f[i - 1] - sum[i]) q.pop_back();//维护队列单调递减
        q.push_back({i,f[i - 1] - sum[i]});
        while(!q.empty() && q.front().tim < i - k) q.pop_front();//删去过期元素
        f[i] = q.front().dat + sum[i];
    }
    printf("%lld", f[n]);
    return 0;
}
