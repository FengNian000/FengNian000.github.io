浅谈KMP

分类：
标签： String

日期：2022-10-04 11:10:06

前两天分了一下学习内容，我分的字符串专项，就回来复习了一下 \(KMP\)。

简介

KMP算法是三位学者在 Brute-Force算法的基础上同时提出的模式匹配的改进算法。Brute- Force算法在模式串中有多个字符和主串中的若干个连续字符比较都相等,但最后一个字符比较不相等时,主串的比较位置需要回退。KMP算法在上述情况下,主串位置不需要回退,从而可以大大提高效率 —— 摘自百度百科

暴力匹配

\(C++\) 的 \(string\) 中也含有能进行字符串匹配的 \(find\) 函数，但它的时间复杂度较高，以下是它的一种实现方式： 

int find(string s, string str) {
    for(int i = 0, j = 0, len = s.size() - str.size() + 1; i < len; ++i) {
        while(s[i + j] == str[j]) ++j;
        if(j == str.size())
            return i;
        else
            j = 0;
    }
    return -1;
}
最坏情况下 \(i\) 会移动 \(n-m+1\) 次而 \(j\) 会在每一次 \(i\) 移动后最坏会移动 \(m-1\)，所以暴力匹配复杂度为 \(O(nm)\)。 虽然 find 足以应对大多数字符串匹配

KMP

很明显，每次匹配失败都会从第一个字符开始匹配，如果这个字符串有形如 a....a 这样的具有前后重复性质的部分就会进行多余的匹配，\(KMP\) 就是一种吸取前面匹配失败的经验进行跳跃式匹配的算法,先看一个视频：

（这里还有一个）

视频中，在 x 失配后因为字符串后面的部分中含有与开头字符串 ab 相同的部分 ,在 x 失配之前两个 ab 都已经在不同的位置匹配完毕，我们可以让第一个 ab 直接去和第二个 ab 所匹配的字符部分去进行接下来的匹配，很明显，视频中在匹配失败后是直接从处在 ab 后面的 c 开始匹配，这样就可以免去前面部分多余的匹配。

nex

可以看出，视频中引入了一个叫做匹配表的东西，我们一般用数组 prefix 或 nex 存储。

这里我们定义 前缀 为字符串的第一个字符往后不包括最后一个字符的子串，后缀 为最后一个字符往前不包括最后一个字符的子串如：

对于字符串 \(aabaaf\) 我们有前后缀：

前缀 后缀
\(a\) \(f\)
\(aa\) \(af\)
\(aab\) \(aaf\)
\(aaba\) \(baaf\)
\(aabaa\) \(abaaf\)

哦对忘了说了 \(nex\) 表示一个字符串的最长公共前后缀 同样，对于字符串 \(aabaaf\) 的 \(nex\) 如下：

子串 最长公共前后缀 \(nex\)
\(a\) \(\emptyset\) \(0\)
\(aa\) \(a\) \(1\)
\(aab\) \(\emptyset\) \(0\)
\(aaba\) \(a\) \(1\)
\(aabaa\) \(aa\) \(2\)

前置芝士完毕，我们看一下 \(nex\) 怎么求: 代码先贴这

void getnext(int nex[], string s) {
    nex[0] = 0;
    for(int i = 1, j = 0; i < s.size(); ++i) {
        while(j && s[j] != s[i]) j = nex[j - 1]; //nex匹配失败,指针j回退
        if(s[i] == s[j]) ++j;                    //i会在for循环增加
        nex[i] = j;
    }
}

这里 \(i\) 在后缀部分上移动， \(j\) 在前缀部分上移动，初始 \(nex[0] = 0\) 不用说，当 \(s_i = s_j\) 时则说明当前成功匹配，让 \(j+1\) ,若当前 \(s_i \neq s_j\) 时则说明匹配失败，我们直接让 \(j\) 回退到上一次匹配成功的地方即 \(nex[j-1]\) ，接着让 \(s_j\) 继续和 &s_i& 比较上述操作，这样我们就得到了 \(KMP\) 的核心数组 —— \(nex\) 表！！

匹配

既然已经求出 \(nex\) 数组，那么我们就可以直接进行匹配了

从别的博客扣的图，凑活看看

int KMP(string s/*文本串*/, string str/*模式串*/) {
    if(str.size() == 0) return -1;
    for(int i = 0, j = 0; i < s.size(); ++i) {
        while(j && s[i] != str[j]) j = nex[j - 1]; //失败，回退
        if(s[i] == str[j]) ++j;                    //同上
        if(j == str.size()) return i - j + 1;
    }
    return -1;
}

匹配部分与求 \(nex\) 部分很像，当 \(s_i \neq str_j\) 就直接让 \(j\) 跳到 \(nex[j-1]\),因为 \(nex[j-1]\) 表示字符子串 \(str_{[0,j-1]}\) 最长的相同前后缀，已经匹配到 \(j\) 就说明，前面 \(str_{[0,j-1]}\) 已经匹配完成，既然 \(j\) 已经失配那么就直接让 \(str_{[0,j-1]}\) 前缀部分和 (后缀部分(已经匹配的部分)) 直接匹配成功,有那么一丢丢绕，我加了括号来分辨。

最后，我们就得到了一篇完整的 \(KMP\) 代码：

#include <bits/stdc++.h>
#define N 100010
using namespace std;
int nex[N];
void getnext(int nex[], string s) {
    for(int i = 1, j = 0; i < s.size(); ++i) {
        while(j && s[j] != s[i]) j = nex[j - 1];
        if(s[i] == s[j]) ++j;
        nex[i] = j;
    }
}
int KMP(string s, string str) {
    if(str.size() == 0) return -1;
    for(int i = 0, j = 0; i < s.size(); ++i) {
        while(j && s[i] != str[j]) j = nex[j - 1];
        if(s[i] == str[j]) ++j;
        if(j == str.size()) return i - j + 1;
    }
    return -1;
}
